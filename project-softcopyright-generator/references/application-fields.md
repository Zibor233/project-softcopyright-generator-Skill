# 申请表字段参考

本文件用于约束软著申请资料中的关键字段来源、字段口径和一致性要求。

## 推荐字段顺序

1. 软件全称
2. 软件简称
3. 版本号
4. 著作权人
5. 开发完成日期
6. 首次发表日期
7. 开发的硬件环境
8. 运行的硬件环境
9. 开发该软件的操作系统
10. 软件开发环境 / 开发工具
11. 该软件的运行平台 / 操作系统
12. 软件运行支撑环境 / 支持软件
13. 编程语言
14. 源程序量
15. 开发目的
16. 面向领域 / 行业
17. 软件的主要功能
18. 技术特点
19. 页数
20. 软件分类

## 字段来源规则

- 软件全称：必须由用户确认，正式资料文件名、系统说明文档标题、代码文档标题和页眉都以该字段为准。
- 软件简称：优先根据软件全称和项目名称建议，由用户确认。
- 版本号：必须由用户确认。如果项目版本号低于 `V1.0`，应提醒用户软著首次提交通常使用 `V1.0`，再由用户决定填写 `V1.0` 或项目当前版本号。
- 著作权人、日期类字段：必须由用户确认，不得擅自硬编码。
- 编程语言、源程序量：根据项目分析结果生成。
- 开发目的、面向领域、主要功能、技术特点：必须优先基于项目业务理解与真实代码整理，不能仅按目录结构泛泛生成。
- 软件开发环境 / 开发工具：只填写 IDE 或编辑器名称，如 `Visual Studio Code`、`Cursor`、`PyCharm`、`IntelliJ IDEA`，不要把 `React`、`Next.js`、`Vite`、`TypeScript` 等技术栈写入该字段。
- 开发该软件的操作系统：填写实际开发电脑操作系统版本，如 `Windows 11`、`Windows 10`、`macOS 14`。
- 该软件的运行平台 / 操作系统：填写软件运行所在的平台或系统，如 `Windows 10/11`、`Linux`、`macOS`。
- 软件运行支撑环境 / 支持软件：填写运行所需的软件环境，如 `Node.js`、`Python`、`MySQL`、`PostgreSQL`、`Redis`、`Docker`、浏览器、中间件或外部服务。
- 开发的硬件环境：优先以当前设备配置作为建议值，不确定时应让用户补充。
- 运行的硬件环境：可默认参考开发硬件环境，但应允许用户按实际部署环境修改。

## 一致性要求

- 软件全称和版本号必须与系统说明文档、代码文档和申请资料中的所有位置保持一致。
- 正式资料生成时，如命令参数中的软件名称或版本号与申请表已确认字段冲突，应以申请表字段为准。
- 主要功能、技术特点、开发目的必须来自当前项目，不得沿用其他范本文案。
- 若存在 `待用户确认` 字段，正式输出前必须尽量替换；若仍保留，需在生成报告中明确提示。
